# PaideiaOS — Nix dev environment

Per `design/02-development-environment.md` §7.1, Nix flakes are the canonical hermetic-build substrate.

## Usage

From the repo root:

```bash
nix develop ./nix
```

That drops you into a shell with all phase-1 tools pinned.

## Layout

```
nix/
├── flake.nix       # primary flake (devShell + toolchain pin)
├── flake.lock      # generated by nix; commit it
├── packages/       # custom derivations (paideia-as, paideia-link, …) — phase 2+
└── shells/         # devShell variants (dev / ci / release) — phase 2+
```

## What's pinned in phase 1

| Tool | Purpose |
|---|---|
| nasm | Phase-1 bootstrap assembler |
| qemu | UEFI-OVMF boot + KVM dev workstation |
| OVMF | UEFI firmware (regular + Secure Boot variants) |
| swtpm | TPM 2.0 emulator (measured boot) |
| xorriso | UEFI-bootable ISO creation |
| mtools | FAT32 manipulation for EFI System Partition |
| gptfdisk | GPT (`sgdisk`) |
| gdb | Debugger (with `-s -S` QEMU stub) |
| clang / lld / gcc / cmake | C toolchain (ACPICA, wasmtime port — phase 2+) |
| python3 | Dev scripts, ACPI dumps |
| git, gh | Version control + GitHub CLI |

## Open items (phase 2+)

- TLA+ tools (TLC, Apalache) — INFRA-O1.
- libFuzzer / AFL++ / honggfuzz — dev-env §9.5.
- Mozilla `rr` (not in nixpkgs as `mozilla-rr`; investigate alternate derivation).
- The `paideia-as` package itself (phase 2 once the assembler is stable enough to pin).

## First-time setup

If you don't have Nix yet, install via the official installer:

```bash
sh <(curl -L https://nixos.org/nix/install) --daemon
```

Then enable flakes (one-time):

```bash
mkdir -p ~/.config/nix
echo "experimental-features = nix-command flakes" >> ~/.config/nix/nix.conf
```

After that, `nix develop ./nix` works from the repo root.
