# v0.18 issue #994 — closure-type grammar and lowering (pa-r18-001)

**Repo**: paideia-os/paideia-as | **Milestone**: v0.18 (#105) | **HEAD**: c43bea7 (4871 tests green)
**Depends on**: pa-r17-004 (LANDED). **Blocks**: #995 (call), #996 (hashmap).

## 1. Scope

**Delivers**:
- New closure *type* syntax `|T1, T2| -> R !{Eff} @{Cap}` distinct from the
  existing fn-ptr type `(T1, T2) -> R !{Eff}`.
- New `NodeKind::TypeClosure` + `TypeData::Closure` (same payload as FnPtr —
  the discriminant is the signal to lower as 16-B fat pointer, not 8-B code).
- Closure *literals* reuse the existing `ExprData::Lambda` (`|x| body` already
  parses via `parse_lambda_pipe`). Type context of the binding site decides
  whether the lambda lowers as fn-ptr or closure. No new expression kind.
- Capture-analysis wired into lowering: `analyze_captures()` output populates
  an env-record layout, allocated on the caller's stack (per AC).
- Runtime shape `Closure { env_ptr: *env, code_ptr: *fn }` (16 B, align 8).
- ≥12 integration tests. Design doc at `design/toolchain/closures.md` (new).

**Defers**:
- Closure *call* mechanics (`mov r14, [f+0]; call [f+8]`) — that's #995.
  #994 lands a callable representation; #995 wires the invocation.
- Heap-allocated env (escape analysis) — v0.20 per AC.
- Generic closures, `impl Fn` trait dispatch, effect-parametric closures.

**Recommendation vs prompt guidance**: the prompt suggested deferring captures.
Do NOT defer — the AC mandates capture-analysis, and the infrastructure is
already present (`capture.rs`, `check_lambda.rs`, wired at
`check_linearity.rs:457`). What's missing is the *env-record synthesis and
lowering plumbing*, not the analysis itself.

## 2. Grammar

Add a top-level dispatch arm in `parse_type_unquantified`
(`crates/paideia-as-parser/src/parse_type.rs` line 189):
```rust
Some(TokenKind::Pipe) => self.parse_type_closure(),
```
New module `parse_type/type_closure.rs` parses:
```
TypeClosure ::= '|' TypeList? '|' '->' Type EffectRow? CapabilitySet?
```
Mirrors the existing FnPtr branch in `parse_type_paren`. Emit
`NodeKind::TypeClosure` with `TypeData::Closure { params, ret, effects,
capabilities }`.

**Disambiguation**: `Pipe` at start of a type is unambiguous — or-patterns
appear only inside match arms, and `|x| body` is dispatched from `parse_expr`,
not `parse_type`.

## 5. IR + emit

**Runtime layout**:
```
struct Closure  { env_ptr: *env; code_ptr: *fn; }  // 16 B, align 8
struct env_rec  { <capture[0]>, <capture[1]>, ... }  // packed by record_layout.rs
```
`env_rec` lives on the *caller's* stack (AC: Phase 1). The code_ptr addresses
a compiler-emitted closure body — a normal function whose implicit 1st arg is
the env_ptr, arriving in **R14** (per calling-convention.md §1 and #995).

**IR**: new `IrKind::ClosureCons` variant. Lower `ExprLambda`-typed-as-closure
to it. Side table `closure_meta` carries env layout + mangled code symbol.

**Emit** (`crates/paideia-as-elaborator/src/emit_lambda.rs`):
- `emit_closure_body` — walks body; captured symbols resolve to `[r14 + off]`,
  params use SysV RDI/RSI/... unchanged.
- `emit_closure_cons` — reserves 16 B for the pair + `sizeof(env_rec)` in
  caller frame; emits `lea rax, [rbp+env_off]; mov [f+0], rax`, populates env
  fields (Consume moves, Reference/Value copies), then `lea rax, [rip +
  closure_<parent>_<node_id>]; mov [f+8], rax`.
- Symbol name mangling: `closure_<parent>_<node_id>`. Reuses #981 addr-of
  relocation machinery.

**Zero-capture case**: still emit the fat pair with env_ptr = 0. This lets
#995 use a uniform `call [f+8]` regardless. Fold-to-fn-ptr optimisation is
future work.

## 7. Backtrack candidates

- **`emit_visit_lambda.rs::visit_lambda`** currently dispatches on lambda body
  shape (App/Action/Match/Store/EnumCons). None recognises a closure literal.
  Likely need a new *outer* arm keyed on the binding's expected type — else
  the closure literal is silently emitted as a bare fn-ptr and the env is
  never populated.
- **local_bindings uses register homes only.** Captured symbols need a memory
  home `[r14 + offset]`. Extend `LocalBindingTable` with a `Home::Env{off}`
  variant (touches every existing lookup site) OR a parallel
  `captured_bindings` map inside `EmitWalker.state`.
- **Ir side-table plumbing**: `arena.captures_mut()` doesn't exist. Follow
  the pattern of `arena.let_meta_mut()` / `arena.record_layout_mut()`.
- **paideia-os canary tests** (`crates/paideia-as/tests/*.rs`) must stay
  green. `action_bodied_lambda_first_instr` is the closest existing lambda
  canary — if we rewrite `visit_lambda`'s outer arm we risk regressing it.
  Add closure arm as a *new* branch.
- **R14 vs bridge saves**: `PAIDEIA_BRIDGE_SAVE = [R15, R14]` already
  preserves R14 across SysV/MS bridges — no bridge amendment needed for #994.
- **Pretty-printer + LSP**: adding `TypeClosure` without pretty-print prints
  `<Type?>` in LSP hover — add early.
