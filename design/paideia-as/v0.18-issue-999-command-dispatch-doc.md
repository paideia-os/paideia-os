## 3. What CAN be delivered today

The substrate landed by v0.17 + v0.18-so-far supports **three** dispatch
shapes end-to-end today. All three should be documented as *current
best-practice* forms, with the hash+closure shape framed as the *target*
form.

### 3.1 Enum-tag dispatch via `match` (cmp/je cascade)

- **Substrate**: unit-variant enums (pa-r17-007), payload-enum matching
  (pa-r17-008), `match` scrutinee lowering.
- **Pattern**: `enum Cmd { Ls, Cd(Str), Cat(Str), Exit }` +
  `match c { Ls => ls_impl(), Cd(p) => cd_impl(p), … }`.
- **Existing witness fixtures**: `pick4_arm_a..d.pdx`, `pick5_arm_a..c.pdx`,
  `pick4uv_arm_*.pdx`, `match_enum_pattern.pdx`.

### 3.2 Enum-tag dispatch via `@jump_table` (O(1) memory-indirect)

- **Substrate**: pa-r15-009 `@jump_table` attribute on `match`
  (landed for **unit-variant enum** scrutinees; witness `pick4uv_arm_*`,
  `pick5uv_arm_*`, `pick8uv_arm_*`). Integer-literal scrutinee jump-table
  codegen was P0273-deferred to pa-r15-009b — confirm status before
  documenting integer form as landed.
- **Pattern**: same enum + `match c @jump_table { … }`.
- **Density contract**: doc must reproduce the pa-r15-009 constraints
  (50% coverage, range ≤ 256, default `_` arm required) — cross-ref, don't
  duplicate.

### 3.3 Function-pointer indirect dispatch

- **Substrate**: fn-ptr types (pa-r17-001 / #979), address-of function
  symbol (pa-r17-004 / #981), indirect call via RIP-relative (#1040).
- **Pattern**: `pub let h : (Str) -> u64 !{} @{} = &ls_impl`; call via
  `(h)(arg)`. Emits `FF 15` byte prefix (verified by
  `pa_r19_1100_gap2_fnptr_indirect_args.pdx` and
  `pa_r17_004c_call_rip_fnptr.pdx`).
- **Composability**: an enum-tagged match can *return* a fn-ptr, or the
  match arms can *call* fn-ptrs — both compose today with landed
  substrate.

**Not yet buildable** (confirm-and-defer, don't guess):
- Fn-ptr **field** of a record (VFS vops table shape) — the design doc
  `fnptr-unsafe-pattern.md` describes it as intended; verify against
  current fixture inventory before claiming it as "today". If missing,
  document as a follow-up (likely trivial once #981 is fully wired).
- Fn-ptr **array** indexed by command-tag u8 — same caveat.
- Anything involving heap allocation of the environment.

## 6. Reference-implementation plan

**One** `.pdx` fixture. Recommended shape:

```pdx
module PaR18006CommandDispatchShell = structure {
  // 4-arm unit-variant command tag — dense enough to exercise the
  // @jump_table O(1) path (density contract satisfied at 100%).
  enum Cmd { Ls, Pwd, Echo, Exit }

  // Handlers — plain fns today, closures tomorrow.
  pub let handle_ls   : () -> u64 !{} @{} = fn(_: ()) -> 1u64
  pub let handle_pwd  : () -> u64 !{} @{} = fn(_: ()) -> 2u64
  pub let handle_echo : () -> u64 !{} @{} = fn(_: ()) -> 3u64
  pub let handle_exit : () -> u64 !{} @{} = fn(_: ()) -> 0u64

  // Dispatch: match on tag → direct call to the corresponding handler.
  // With @jump_table, this is O(1) memory-indirect once codegen lands
  // for unit-variant enums (which it has per pick4uv_*).
  pub let dispatch : (Cmd) -> u64 !{} @{} = fn(c: Cmd) -> match c @jump_table {
    Ls   => handle_ls(()),
    Pwd  => handle_pwd(()),
    Echo => handle_echo(()),
    Exit => handle_exit(()),
  }

  pub let entry : () -> u64 !{} @{} = fn(_: ()) -> dispatch(Cmd::Echo)
}
```

**Runtime canary**: follows the `pick4uv_arm_a.rs` shape — `RuntimeCase` +
`run_and_verify`, expected exit code `3`.

**Why exactly one fixture**: the doc's job is to canonicalise the shape,
not to re-prove pa-r15-009 or pa-r17-004. Every additional fixture
duplicates a landed canary. If the doc reader needs to see an alternative
form (cmp/je cascade instead of `@jump_table`, fn-ptr indirect instead of
direct call), the doc cross-refs the existing `pick4_arm_*` and
`pa_r17_004c_call_rip_fnptr` fixtures rather than duplicating them.

**Adversarial verification hook** (per version-discipline memory):
the workerbee handoff must confirm `cargo test -p paideia-as --test
runtime command_dispatch_shell` returns exit code 3, AND that opcode-canary
pre-push gate stays green. Neither claim can be taken from workerbee's
summary alone; debugger re-runs both.

## 9. Backtrack candidates

1. **`@jump_table` on payload-carrying enum variants** may still fall
   back to cmp/je cascade in current impl (the landed proof is
   `pick4uv_*` = unit variants; `pick4_arm_*` = payload but *without*
   `@jump_table`). The reference fixture in §6 uses unit-variant only for
   exactly this reason. If workerbee wants to demo `Cd(Str)` payload with
   `@jump_table`, verify pa-r15-009b status first — do NOT ship a
   silently-cmp-je fixture with a doc that claims jump-table.
2. **`Str` as a function parameter type**. Confirmed via #998 (Str type
   landed, downgraded). The fixture in §6 sidesteps this by using unit
   variants only — safer. If richer example is wanted, cross-ref
   `v0.18-issue-998-string-str-stdlib.md` for what Str shape survived
   the downgrade.
3. **`enum Cmd::Ls` construction syntax**. Confirm the fixture uses the
   qualified form (`Cmd::Echo`) not bare (`Echo`) — bare works in
   `pick*_arm_*` because there's a single enum in scope, but in this doc
   we recommend qualified form for hygiene. If elaborator rejects
   `Cmd::Echo` (it shouldn't — `enum_cons_result_ok.pdx` uses
   `Result::Ok`), fall back to bare.
4. **`match c @jump_table { … }` position**. In the landed grammar per
   `jump-table-attribute.md` §Grammar the attribute appears *after* the
   scrutinee and *before* the brace. The fixture in §6 follows this; do
   not accept a workerbee draft that puts it elsewhere.
5. **paideia-os canary regression risk**. This issue adds only new
   files; no changes to `crates/paideia-as/src/` or public surface. The
   paideia-os canary set (`crates/paideia-as/tests/*.rs`) must stay
   green — trivially true for a doc-plus-fixture change, but the
   pre-push find-paideia-as.sh must still pass.
6. **Cross-repo footprint**. This is a paideia-as-repo doc that
   *unblocks* paideia-os phase 11 planning; paideia-os itself takes no
   action from #999 landing beyond an eventual doc cross-ref. No
   submodule bump is *required* by #999, but if the doc is referenced
   from a paideia-os design doc in the same window, the standard bump
   flow applies.
7. **Version discipline**. Mid-release addition per
   `feedback_paideia_as_version_discipline.md`: CHANGELOG entry under
   `## [Unreleased] v0.18` only. No `workspace.version` bump, no tag —
   those move at v0.18 close.
