## 1. Scope

**Delivers**:
- Canonical `struct Str { ptr: *u8, len: u64 }` in `crates/paideia-stdlib/pdx/str.pdx` (16-byte record; `*T` decoded as 8-byte pointer by `struct_registry::decode_field_type`).
- Module-level Str constant pattern (end-to-end): `let hello_bytes : [u8; 6] = "hello\0"` + `let hello_str : Str = Str { ptr: &hello_bytes, len: 5u64 }`. Emits 16 bytes in .rodata with one W64 relocation at field offset 0. Verified path: #1074 (Borrow inside RecordCons populates AddrOfSideTable) + #1157 (encode_record_cons emits width-correct field bytes) + `cmd_build.rs:1795–1874` (RecordCons Borrow field → RelocSpec at field offset).
- Field-read helpers: `str_hello_len : () -> u64` reads `hello_str.len` via RIP-relative (module symbol path); `str_len_of : (*Str) -> u64 = fn(s: *Str) -> s.len` reads via `FieldAccess(Deref(Var))` from [RDI + 8]. Both are established shapes with existing test coverage (`field_read_multi_field.pdx`, `module_field_read_tail.pdx`).
- One **unsafe-asm-backed byte accessor** as escape hatch: `str_byte_at : (*Str, u64) -> u8` — loads `[RDI+0]` into RAX (the ptr), then `movzx eax, byte [rax + rsi]`. Unlocks per-index byte reads today without waiting for expression-position `*p` deref.
- 16 fixture files under `crates/paideia-stdlib/pdx/` — mix of runnable end-to-end and parse-only API surface (see §5).
- 2 runtime canaries under `crates/paideia-as/tests/runtime/str/`: `str_hello_len_returns_5` and `str_byte_at_zero_returns_h` (0x68 = 'h').
- CHANGELOG entry under v0.18.

**Defers** (with follow-up issue proposals — see §7):
- `str_eq`, `str_hash`, `str_substring`, `str_split_at` in their real byte-iterating form. Blocked on the four gaps below.
- Passing `Str` by value (two 8-byte fields, would classify to INTEGER+INTEGER in SysV). Today the pair-form marshalling in `emit_call.rs:75` is enum-specific (discriminant+payload) and pos-0-only (`emit_call.rs:509`). File **#998d**.
- Expression-position deref `*p` yielding a scalar in a register. Today `IrKind::Deref` is only recognised as part of the `FieldAccess(Deref(Var))` shape (`emit_field_access.rs:528`). File **#998b**.
- High-level `while`/`for` in expression position. `ExprLoop` / `ExprFor` parse but lower to `IrKind::Action` (no-op) per `lower/kind_map.rs:77`. `while` INSIDE an `unsafe { block: { … } }` works (`pa_r17_unsafe_while_stmt.pdx`) but that is raw asm, not stdlib. File **#998c**.
- Returning a `Str` by value from a function. Two-slot aggregate return has no emit path today (see MS-ABI T-diag catalog `catalog.toml:1586`: "aggregate returns" are gated out). File **#998e**.
- `String` (owned/growable). Requires a heap allocator — the arena/box/alloc stdlib fixtures are parse-only trait stubs today; there is no working heap. Not in #998 AC. Track under `pa-r18-003+` follow-ups.

## 2. Prerequisite check — verdict: **PROCEED with reduced runtime scope**

String-literal parsing and lowering is **not** blocked:

| Layer         | Status | Evidence                                                                    |
|---------------|--------|-----------------------------------------------------------------------------|
| Lexer         | ✓      | `TokenKind::StringLit` + `scan_string.rs` (7+ passing tests)                 |
| Parser        | ✓      | `parse_primary.rs:81` → `ExprData::StringLiteral(String)`                    |
| AST → IR      | ✓      | `lower/kind_map.rs:25` `ExprString → IrKind::StringLiteral`                  |
| IR side-table | ✓      | `literal_bytes.rs`, `string_literal.rs`, `string_intern` (FNV-1a)            |
| .rodata emit  | ✓      | `data_encoder.rs:266–281` StringLiteral → interned `__str_...` reloc         |
| Let RHS       | ✓      | `pa_r12_001_string_literal_module.pdx` (`let X : [u8;N] = "..."` in-file bytes) |
| **RecordCons Borrow field** | ✓ | #988v2 / #1074 / #1157 chain — verified path for `Str { ptr: &bytes, len }` |
| **FieldAccess on (*Str)** | ✓ | `emit_field_access.rs:480–617` — `FieldAccess(Deref(Var))` supports u8/u16/u32/u64/i8..i64/*T |

What is **absent** that would upgrade the layer from "type + accessors + parse-only surface" to "full comparison/substring/split/hash":

1. Expression-position `*ptr` scalar load (#998b).
2. High-level `while` / `for` outside unsafe (#998c).
3. Struct-by-value arg passing beyond pos-0 pair-form enums (#998d).
4. Struct-by-value return (#998e).

Conclusion: the string-literal pipeline is real and testable end-to-end for **construction and field access** — the shape needed by kernel commands that hold table-lookup keys as constants. The full stdlib API surface documented here shipped as parse-only + concrete field-access helpers is honest to what the compiler supports today. Follow the #997 pattern: the aspirational trait/surface lives in `str.pdx`; the runnable substance lives in per-shape fixture files.

## 3. Representation

```
struct Str { ptr: *u8, len: u64 }

byte layout (per record_layout tight-pack, natural alignment):
  offset 0: ptr  (8 bytes, W64 pointer/relocation)
  offset 8: len  (8 bytes, u64 little-endian)
  total size: 16, align: 8
```

Chosen over length-prefixed for three reasons: (a) `*u8` field size 8 is already recognised by `decode_field_type` (`struct_registry.rs:252`), no new field-type infrastructure needed; (b) the `&bytes_array` → relocation at field offset 0 path is already wired end-to-end; (c) matches Rust's `&str` / Go's `string` / C++ `string_view` — familiar to kernel authors on paideia-os.

The **canonical fixture pattern** for constant strings:
```pdx
let hello_bytes : [u8; 6] = "hello\0"                       // .rodata, self-contained bytes
let hello_str   : Str      = Str { ptr: &hello_bytes, len: 5u64 }  // .rodata, 16 bytes + 1 reloc
```

For runtime-produced strings (kernel input buffers), callers hold a `*Str` and read `.ptr` / `.len` via the `FieldAccess(Deref(Var))` path.

## 7. Backtrack candidates

1. **`&globalArray` in RecordCons field fails to elaborate.** Confidence: MEDIUM-HIGH. The path is wired for `struct { fnptr_field }` per #988v2 and the shim for record-field-Borrow at `cmd_build.rs:1303–1336` iterates all Borrow children of RecordCons — but the only production test for it is fnptr-typed. Fallback: file **#998a** "RecordCons Borrow field targeting data symbol (array/global)" and downgrade the module Str constants to placeholder-only (leave the type + ptr-form field access).
2. **`*u8` in struct-field decoding.** Confidence: HIGH — `decode_field_type` explicitly handles `s if s.starts_with('*')`.
3. **`str_bytes_yields.pdx` / `string_from_str.pdx` name collision** (they redeclare `Str` with `data:` not `ptr:`). Confidence: HIGH — MODIFIED entries in §4 rename `data → ptr` to align on the canonical shape. Sub-risk: cross-file `Str` name collisions if kernel already uses `Str` differently; the module-scoped struct declarations sidestep this (each `.pdx` declares its own).
4. **Unsafe-block byte load in `str_byte_at`.** The `unsafe { … block: { … } }` shape with function parameters bound to RDI/RSI is the SysV default convention; confirmed via `cap_smoke.pdx` (RBX baseline) and #1200 (RSI-second-arg). Confidence: HIGH.
5. **`build.rs` plumbing.** `paideia-stdlib/build.rs` only regenerates the UEFI header template; no changes needed.
6. **`parse_pdx.rs` `#[ignore]` posture.** Continue #997 posture — every new parse test is `#[ignore]`d.
