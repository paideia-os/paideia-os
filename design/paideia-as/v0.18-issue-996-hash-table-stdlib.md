**Repo**: paideia-os/paideia-as | **Milestone**: v0.18 (#105) | **HEAD**: 19a536f (4942 tests green)

## 1. Scope

The AC target — "`HashMap<K, V>` … `!{Alloc}` when the load factor forces a rehash; 24 unit tests including collision cascades, tombstones, rehash under load-factor threshold" — presumes generic monomorphization (#997c, deferred), a working heap allocator (no substrate today), and Str-keyed dispatch (blocked on #1234). None of that stack exists. Ship the shape that lands cleanly on the substrate we have, and split the AC into concrete follow-ups. This mirrors the #998 downgrade discipline.

**Delivers**:
- Canonical `HashMapU64U64` — fixed-capacity, open-addressing (linear probing), u64→u64. 32 slots (5 bits), power-of-two so mask-modulo is cheap. Module-scope mutable buffer in `.bss`.
- `hashmap_u64_u64.pdx` in `crates/paideia-stdlib/pdx/` holding the canonical shape + doc surface. Bucket layout: `[key:u64, value:u64]` per slot (16 B), tombstone marker via `state:u64` sentinel column parallel to key column (0 = empty, 1 = occupied, 2 = tombstone). Total buffer: `[u64; 96]` (32 slots × 3 words).
- Operations `hashmap_u64_new`, `hashmap_u64_put`, `hashmap_u64_get`, `hashmap_u64_contains`, `hashmap_u64_len` — bodies are `unsafe { block: {...} }` blocks because expression-position array indexing lowers to `ExprCall` phase-1 (`parse_postfix.rs`), i.e. there is no non-asm indexing surface today. `get` returns `OptionU64` (via #997).
- Hash function: Fibonacci-style `k ⊕ (k >> 33)`, then `imul k, 0x9E3779B97F4A7C15` (golden-ratio prime), then `shr rax, 59` to fold into 5-bit index. No `rol` needed; `rol` gate (pa-r15-004) becomes irrelevant.
- 5 driver fixtures in `tests/build-emit/` + 5 runtime canaries in `crates/paideia-as/tests/runtime/hashmap/`.
- CHANGELOG entry under `## [Unreleased] v0.18`.

**Defers** (with follow-up issue proposals):
- `HashMapStrU64` — Str-keyed. Blocked on #1234 (RecordCons Borrow of data symbol) which prevents module-scope Str constants, plus #998b/c for byte-iterating hash. File **#996b** "HashMap with Str keys — command-name dispatch shape".
- `HashMapU64Closure` — closure-valued (16 B fat pair per slot). No blocker beyond widening the value column from `[u64]` to `[u64; 2]` and ClosureCall wiring; but the actual consumer is #1006 (paideia-os phase-11 shell), not v0.18. File **#996c** "HashMap with closure values — command dispatch payload".
- `remove` operation. Not needed by any current v0.18 consumer; tombstone-marking probe is designed for it (state column above) so the follow-up is a ~30-LOC addition. File **#996d** "HashMap::remove + tombstone scan".
- Heap-backed variable-capacity + rehash. Blocked on the entire absent allocator substrate (see `crates/paideia-stdlib/pdx/alloc.pdx` — parse-only trait stubs, no working `!{Alloc}` handler). File **#996e** "HashMap heap-backed rehash — needs pa-r18-014+ alloc".
- Robin-hood displacement tracking. Zero benefit at fixed capacity + linear probing; correct at load ≤ 0.5, i.e. ≤ 16 live entries. File **#996f** "Robin-hood probe swap on heap-backed variant".
- AVX2 parallel probe. Already scoped as pa-r18-011 (#1004). Not touched here.
- `iter` / range API. No `for` in expression position (#998c). File **#996g** contingent on #998c.

**Recommended monomorphization**: **HashMapU64U64 only** for MVP. Str keys are the shape the doc AC actually wants (command-name dispatch, per the #999 canonical shape), but that path stays broken until #1234 lands. Closure values are the second-most-valuable form and land in #996c once someone actually consumes them. Numeric MVP is the unblocked cut.

**Not in scope**: any modification to `paideia-as-encoder`, elaborator, parser, or emit paths. Additive-only: one `.pdx` file, N fixtures, canaries, parse tests, one CHANGELOG line.

## 3. Representation

```
Module-scope buckets buffer (writable .bss, 32-slot capacity):

  let mut hm_state  : [u64; 32] = uninit    // 256 B — 0 empty / 1 occupied / 2 tombstone
  let mut hm_keys   : [u64; 32] = uninit    // 256 B
  let mut hm_values : [u64; 32] = uninit    // 256 B
  let mut hm_len    : u64       = 0         //   8 B

Struct-of-arrays chosen over array-of-struct (single 96-word `[u64;96]`):
  (a) state column is the probe-hot loop — separating it fits it entirely into 1-2
      cache lines for the whole table;
  (b) simpler asm — `[hm_state + rsi*8]` vs `[hm_buckets + rsi*24]` (scale 24
      is not a legal SIB scale, would need `imul rsi, 24` first);
  (c) tombstone rewrite for #996d touches only hm_state.

Hash → slot index (Fibonacci hash, 32 slots, mask = 0x1F):
  h  = k ⊕ (k >> 33)
  h  = h × 0x9E3779B97F4A7C15    ; imul r64, imm64 (Golden ratio prime)
  h  = h >> 59                    ; fold into 5-bit index
  ix = h                          ; already ≤ 31; no explicit mask
```

Capacity 32 chosen for (a) small enough to fit whole table in one 4 KB page, (b) power of two → shift instead of div, (c) load-factor budget of 16 for correctness (all callers today have < 8 entries). Follow-up #996e swaps for a heap-backed variant with dynamic sizing.

## 5. API surface

```pdx

struct HashMapU64U64 { }  // opaque module-scope singleton (no per-instance layout today)

// Documented API (real bodies in hashmap_u64_u64.pdx):
//   fn hashmap_u64_new()                  -> () !{} @{}                // resets state, len ← 0
//   fn hashmap_u64_put(k: u64, v: u64)    -> u64 !{} @{}               // returns 1 on insert/replace, 0 on full
```

`hashmap_u64_u64.pdx` inlines the buffer + all fn bodies. Each function body is a single `unsafe { block: { ... } }` with SysV-conventional arg mapping (RDI = key, RSI = value on `put`).

**Return convention**: `get` returns an `OptionU64` (16 B pair — discriminant + payload) using the existing enum-pair emit path (`emit_call.rs:75`, pos-0 pair form) that #997 already exercises. Callers pattern-match; canaries drive `hashmap_u64_unwrap_or_zero` = `match get(k) { Some(v) => v, None => 0 }`.

## 7. Backtrack candidates

1. **`let mut arr : [u64; N] = uninit` at module scope emits as writable .bss, not .rodata.** Confidence: HIGH. `pa_r13_008_mut_array_uninit.pdx` is the exact shape; `cap_table_addr.pdx` proves the reloc path. If workerbee reports the array lands read-only, check `cmd_build.rs` SectionKind selection for `mut` array literals — do not silently downgrade to a smaller non-mut buffer.
2. **`imul r64, r64, imm64` with a 64-bit immediate.** iced-x86 encodes it as `48 69 /r imm32` (sign-extended imm32, NOT full imm64). The golden-ratio constant `0x9E3779B97F4A7C15` does NOT fit in imm32. Mitigation: `mov rax, 0x9E3779B97F4A7C15; imul rdi, rax` — two instructions, no encoding gotcha.
3. **SIB scale of 24 for a packed bucket layout.** Not a legal x86 scale (1/2/4/8 only). This is why the design is struct-of-arrays; a single-buffer AoS variant would require pre-shifting the index. Fixed by design.
4. **`OptionU64` return from an unsafe-asm-bodied function.** Return-value ABI for enum pair is discriminant in RAX, payload in RDX (#997 shape). The unsafe block must produce both. Confidence: MEDIUM-HIGH — the wire ABI is documented but this is the first stdlib fn returning an enum pair from unsafe asm. Backup: return `u64` sentinel (0 = None, any other = value) and adapt callers, but that regresses the #997 integration. Verify with a scratch `.pdx` before shipping.
5. **Cross-fixture buffer state leakage.** Each fixture is compiled/linked/run independently, so the module-scope buffer is fresh per canary invocation. `hashmap_u64_new()` still zeroes state explicitly (defensive; also proves reset works). If workerbee omits the explicit zero, canary ordering could hide bugs.
6. **`bool` return type for `contains`.** #1229 lowered comparison ops end-to-end but `bool` as a return-value ABI hasn't been exercised in every emit path. Downgrade to `u64` return (1/0) per §5 note; upgrade to `bool` in #996h once #1229 has been proven with a bool-return canary.
7. **Raw `cmp / jne / jmp` probe loop with a local label (`probe_loop:`) inside unsafe block.** `ahci_fis_stub.pdx` proves the label + jcc pattern works. Confidence: HIGH.
8. **`match opt { Some(v) => v, None => 0 }` composed with the unsafe-asm `get`.** The RAX/RDX pair return path exists but the composition of "unsafe-asm callee returns enum pair; safe caller pattern-matches" has no landed witness. Verify with a bare `option_from_unsafe.pdx` scratch fixture before writing the full collision canary.
9. **Encoder round-trip on `mov r64, [rip + symbol + rsi*8]`.** iced-x86 requires `[rip + disp32]` (no index in RIP-relative). Mitigation: `lea rax, [rip + hm_state]` first, then `mov rdx, [rax + rsi*8]`. Two-step is standard PIC-safe code. Do not accept a single-op form from workerbee — it is not encodable.
10. **Version discipline.** Mid-release addition per `feedback_paideia_as_version_discipline.md`: CHANGELOG under `## [Unreleased] v0.18` only. No workspace.version bump or git tag — those move at v0.18 close.
