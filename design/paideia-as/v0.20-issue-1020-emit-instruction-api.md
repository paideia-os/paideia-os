# 1. Scope

Land the **first stable public entry point** on the `paideia-as-runtime` crate:

```rust
pub fn emit_instruction(buf: &mut CodeBuffer, ins: Instruction) -> Result<(), EmitError>;
```

Downstream consumer #1 is the paideia-os phase-10 WASM jail: it will emit small
instruction sequences at runtime into a JIT page. Downstream consumer #2 is
paideia-as's own self-hosting path (v0.21+). Both need a **shape that will not
change under them** — the internal `encode_instruction(&Instruction, &mut CodeBuffer, &mut EncodeStats) -> Result<EncodeOutput, EncodeError>`
signature is not stable (it leaks tightening stats and reloc/label-fixup lists that
belong to the batch-linker pipeline, not to a single-instruction JIT emit).

**In scope:**

- One new module `crates/paideia-as-runtime/src/emit.rs`.
- One new integration-test file `crates/paideia-as-runtime/tests/emit_api.rs`
  with **12 tests** (6 success, 4 error, 2 discipline).
- A Cargo **feature `emit`** on `paideia-as-runtime` gating the encoder
  dependency (see §3.3 for the no_std-preservation rationale).
- Documented mapping from `paideia_as_encoder::EncodeError` → runtime
  `EmitError`.
- `CodeBuffer` re-exported from `paideia_as_runtime::emit` so callers link one
  path.

**Out of scope (belongs to later r20 issues):**

- Symbol resolution — pa-r20-006 owns `resolve_symbols`. `emit_instruction`
  refuses reloc-producing operands with `EmitError::UnresolvedRelocation` and
  rolls the buffer back.
- Generic `Buf: Write` trait — recorded as backtrack §7.3; may land in r21 when
  the WASM jail reports concrete needs (raw JIT pages, ring buffers).
- endbr64/xsave (pa-r20-003 / pa-r20-004) — orthogonal encoder issues.
- Migrating the encoder itself to `no_std` — pa-r20-001b, currently deferred
  (v2 §2.3 of #1019 codifies the "encoder stays std" posture).

# 2. Design premises drawn from the current tree at HEAD 28c3c2c

## 2.1 Encoder's current single-instruction shape

`paideia-as-encoder/src/encode_instruction.rs:281`:

```rust
pub fn encode_instruction(
    inst: &Instruction,
    buf: &mut CodeBuffer,
    stats: &mut EncodeStats,
) -> Result<EncodeOutput, EncodeError>
```

- **Takes `&Instruction`**, not owned. #1020 AC takes owned. The wrapper adds
  a borrow.
- **Returns `EncodeOutput`** with `reloc_sites: Vec<RelocSite>` and
  `label_fixups: Vec<LabelFixup>`. These belong to the object-file pipeline
  (elaborator + linker). A JIT caller emitting a self-contained instruction
  gets an empty `EncodeOutput` — the runtime wrapper enforces this and refuses
  otherwise.
- **Takes `&mut EncodeStats`**, which counts total instructions and tightening
  events. Diagnostic-only. The runtime wrapper allocates a throwaway
  `EncodeStats` internally.
- **Error type `EncodeError`** has four variants (`OperandCount`,
  `OperandShape`, `InvalidOperand(&'static str)`, `Unsupported(&'static str)`)
  and is `#[derive(thiserror::Error)]`. The `&'static str` payloads leak encoder
  internals (e.g., `"invalid register id"`, `"Operand::Var reached encoder"`).
  The runtime wrapper collapses them behind a **non-exhaustive** enum whose
  message text is owned by runtime.

## 2.2 `CodeBuffer` is the concrete buffer

`paideia-as-encoder/src/encode.rs:139`:

```rust
#[derive(Default, Debug)]
pub struct CodeBuffer {
    pub bytes: Vec<u8>,
}

impl CodeBuffer {
    pub fn new() -> Self { … }
    pub fn len(&self) -> usize { … }
    pub fn is_empty(&self) -> bool { … }
    pub fn as_slice(&self) -> &[u8] { … }
}
```

The tactical roadmap (`design/roadmap/paideia-as-tactical-issues.md:186, 2291`)
names the parameter `Buf`; the issue AC (`gh issue view 1020`) names the type
`CodeBuffer`. **The roadmap shorthand does not mean a trait — the AC binds the
concrete type.** §7.3 documents the trait as a named-and-owned backtrack for a
later issue if a concrete non-`CodeBuffer` consumer surfaces.

## 2.3 Runtime crate's current dependency posture (post-#1019)

`crates/paideia-as-runtime/Cargo.toml` at 28c3c2c:

```toml
[dependencies]
smallvec = { workspace = true }
static_assertions = { workspace = true }

[dev-dependencies]
paideia-as-encoder = { path = "../paideia-as-encoder" }
paideia-as-ir = { path = "../paideia-as-ir" }
iced-x86 = { workspace = true }
```

- Runtime is `#![no_std] extern crate alloc;` (`src/lib.rs:8-9`).
- Encoder is **not** a runtime `[dependencies]` entry — only `[dev-dependencies]`.
- The #1019 v2 verification recipe asserts
  `cargo check --target x86_64-unknown-none -p paideia-as-runtime` passes.

**#1020's move must preserve that check** — that is the load-bearing invariant
of the runtime crate's identity. §3.3 shows how a Cargo feature gate does this
without moving the encoder.

## 2.4 Reloc-producing operand variants

`paideia-as-runtime/src/instruction.rs:722-808` defines `Operand` with three
variants that today's encoder path resolves into `RelocSite` /
`LabelFixup` entries in `EncodeOutput`:

- `Operand::SymbolRef { name, addend }` (rel32 / abs32 / abs64 depending on
  mnemonic).
- `Operand::LabelRef { name, addend }` (rel8 / rel32 within a function).
- `Operand::MemRipRelSym { name, addend }` (rel32).
- `Operand::MemSymIndexed { name, addend, index, scale }` (abs32).

And one unresolved-var variant:

- `Operand::Var { name }` — should never reach the encoder; hits an
  `unreachable!` in encode_instruction.rs:687/2244. Runtime traps this
  **before** dispatch (see §4.4).

These are the pattern-match keys for `EmitError::UnresolvedRelocation` and
`EmitError::InvalidOperand`.

# 3. API signature and shape

## 3.1 The one public function

```rust
/// Emit a single instruction's bytes into `buf`.
///
/// On `Ok(())`, the buffer's `bytes` vector has been extended with a fully
/// self-contained encoding of `ins`. The buffer's prior contents are
/// untouched.
///
/// On `Err(_)`, the buffer is left **exactly as it was on entry** (length and
/// contents restored via truncate). Callers may safely reuse `buf` after an
/// error without needing to reset it themselves.
///
/// # Errors
///
/// See [`EmitError`] for the full taxonomy. In particular, this function
/// refuses instructions whose operands would produce relocations
/// (`Operand::SymbolRef`, `Operand::LabelRef`, `Operand::MemRipRelSym`,
/// `Operand::MemSymIndexed`) — those must be routed through
/// [`crate::resolve::resolve_symbols`] when it lands in pa-r20-006.
///
/// # Stability
///
/// This function is part of the paideia-as-runtime v0.20 stable public API.
/// Signature changes require a major-version bump; adding variants to
/// [`EmitError`] is compatible because the enum is `#[non_exhaustive]`.
pub fn emit_instruction(
    buf: &mut CodeBuffer,
    ins: Instruction,
) -> Result<(), EmitError>;
```

**Ownership choice.** Owned `Instruction` (not `&Instruction`) matches the AC
verbatim and closes a real footgun: JIT callers construct one Instruction, emit
it, then drop it. Borrowing invites callers to hold a `&Instruction` across an
emit, which is fine but reads awkwardly with the "fire and forget" JIT pattern.
Internally the wrapper takes `&ins` for the encoder call; the `ins` value is
consumed at function boundary.

## 3.2 The re-exported `CodeBuffer`

```rust
// In paideia-as-runtime/src/emit.rs:
pub use paideia_as_encoder::CodeBuffer;
```

This is the **single load-bearing path** callers link. Downstream (paideia-os
WASM jail) writes:

```rust
use paideia_as_runtime::{CodeBuffer, EmitError, Instruction, emit_instruction};
```

Callers never need to name `paideia_as_encoder` directly.

## 3.3 Where the API lives — `paideia-as-runtime`, gated on a Cargo feature

The AC ("Touching files: paideia-as-runtime/") and the milestone one-liner
("linkable library with stable public API (emit_instruction, encode_symbol)")
put this in runtime. But runtime's #1019 v2 verification pins the no_std posture.

adds one new step for the default-features check.

## 3.4 What the wrapper does internally

Pseudocode of `emit_instruction`:

```rust
pub fn emit_instruction(
    buf: &mut CodeBuffer,
    ins: Instruction,
) -> Result<(), EmitError> {
    // 1. Pre-flight: reject Var operands + reloc-producing operands here so
    //    that we never let the encoder write placeholder bytes we'd have to
    //    roll back. Cheap: single pass over inst.operands.
    for op in &ins.operands {
        if matches!(op, Operand::Var { .. }) {
            return Err(EmitError::InvalidOperand);
        }
        if matches!(
            op,
            Operand::SymbolRef { .. }
                | Operand::LabelRef { .. }
                | Operand::MemRipRelSym { .. }
                | Operand::MemSymIndexed { .. }
        ) {
            return Err(EmitError::UnresolvedRelocation);
        }
    }

    // 2. Snapshot buffer length for rollback on encoder failure.
    let saved_len = buf.bytes.len();

    // 3. Delegate to encoder. Discard EncodeStats (throwaway) and check
    //    EncodeOutput has no reloc/label fixups (belt-and-braces given §1
    //    pre-flight, but reloc emit can happen at inner-encoder level from
    //    encoding hints we did not statically catch — better safe).
    let mut stats = paideia_as_encoder::EncodeStats::new();
    match paideia_as_encoder::encode_instruction(&ins, buf, &mut stats) {
        Ok(output) => {
            if !output.reloc_sites.is_empty() || !output.label_fixups.is_empty() {
                buf.bytes.truncate(saved_len);
                return Err(EmitError::UnresolvedRelocation);
            }
            Ok(())
        }
        Err(e) => {
            buf.bytes.truncate(saved_len);
            Err(EmitError::from(e))
        }
    }
}
```

Two rollback triggers, one enum path. `buf.bytes.truncate(saved_len)` is the
**one line that makes the "buffer unchanged on Err" contract cheap** — the
encoder wrote partial bytes into `buf.bytes` (a `Vec<u8>`), and truncate
restores it in O(1).

# 4. `EmitError` — the stable public enum

```rust
/// Errors returned by [`emit_instruction`].
///
/// This enum is `#[non_exhaustive]` so that future paideia-as-runtime releases
/// may add variants without a semver bump on downstream users. Match on it
/// with a wildcard arm.
#[derive(Clone, Debug, PartialEq, Eq)]
#[non_exhaustive]
pub enum EmitError {
    /// The instruction's operand count did not match its mnemonic.
    OperandCount {
        /// The mnemonic that failed the arity check.
        mnemonic: Mnemonic,
        /// Number of operands the mnemonic accepts.
        expected: usize,
        /// Number of operands supplied.
        got: usize,
    },

    /// The instruction's operand shape (register / immediate / memory form
    /// combination) did not match its mnemonic's supported forms.
    OperandShape {
        /// The mnemonic that failed the shape check.
        mnemonic: Mnemonic,
    },

    /// An operand contained a value that cannot be encoded — for example, a
    /// [`Operand::Var`] that was not resolved to a concrete register before
    /// emit, or a register id outside the 0..15 range.
    InvalidOperand,

    /// The instruction, mnemonic form, or CPU-feature requirement is not
    /// supported by this build of the encoder.
    Unsupported,

    /// The instruction referenced an unresolved external symbol, label, or
    /// symbol-relative memory operand. `emit_instruction` emits only
    /// self-contained instructions; route these through the
    /// pa-r20-006 `resolve_symbols` API instead.
    UnresolvedRelocation,
}
```

## 4.1 `From<EncodeError>` mapping

Inside `emit.rs`, private:

```rust
impl From<paideia_as_encoder::EncodeError> for EmitError {
    fn from(e: paideia_as_encoder::EncodeError) -> Self {
        use paideia_as_encoder::EncodeError as E;
        match e {
            E::OperandCount { mnemonic, expected, got } =>
                EmitError::OperandCount { mnemonic, expected, got },
            E::OperandShape { mnemonic } =>
                EmitError::OperandShape { mnemonic },
            E::InvalidOperand(_msg) => EmitError::InvalidOperand,
            E::Unsupported(_msg) => EmitError::Unsupported,
        }
    }
}
```

The `&'static str` payload from encoder is discarded on purpose: encoder tunes
those strings across releases (they read as debug hints, not as documented
error text). Runtime's stability contract does not extend to strings encoder
authors expect to reword.

## 4.2 `Display` and `Error` impls

Runtime should not pull `thiserror` — that adds a proc-macro dep across the
`#![no_std]` root. Hand-write `core::fmt::Display` and gate
`std::error::Error` behind `#[cfg(feature = "std")]` if it turns out to be
needed. The `emit` feature does **not** transitively imply `std` today — but
because `paideia-as-encoder` (a `[dependencies]` under `emit`) is std-flavored,
the resulting runtime crate binary is std when `emit` is on. Concretely: emit=on
consumers get `std` transitively; emit=off consumers stay no_std. Adding a
runtime-owned `std::error::Error` impl gated on `#[cfg(feature = "emit")]` is
therefore safe and free.

```rust
impl core::fmt::Display for EmitError {
    fn fmt(&self, f: &mut core::fmt::Formatter<'_>) -> core::fmt::Result {
        match self {
            EmitError::OperandCount { mnemonic, expected, got } =>
                write!(f, "operand count mismatch for {:?}: expected {}, got {}",
                       mnemonic, expected, got),
            EmitError::OperandShape { mnemonic } =>
                write!(f, "operand shape mismatch for {:?}", mnemonic),
            EmitError::InvalidOperand =>
                f.write_str("invalid operand for emit"),
            EmitError::Unsupported =>
                f.write_str("mnemonic or operand form unsupported in this build"),
            EmitError::UnresolvedRelocation =>
                f.write_str("instruction requires symbol/label resolution; use resolve_symbols"),
        }
    }
}

#[cfg(feature = "emit")]
impl std::error::Error for EmitError {}
```

## 4.3 Why non_exhaustive

Downstream callers (WASM jail, paideia-pq-sign integration) match `EmitError`
with a wildcard arm. Adding a variant (e.g., `RegisterOutOfRange`,
`SegmentPrefixInvalid`) then requires no downstream change. Encoder churn
lands as new `Unsupported` cases here in the interim.

## 4.4 The pre-flight vs. delegate contract

**Pre-flight catches only operand-level unresolved-relocation and Var.** All
other errors — count, shape, sizing — fall through to the encoder, which
returns them uniformly. This keeps the wrapper's control flow flat: one loop,
one match. Adding "clever" pre-flight for arity here would duplicate encoder
knowledge and drift.

# 7. Backtrack candidates

Named-and-owned, each with a specific trigger.

## 7.1 `paideia-as-runtime` cannot host the encoder dep even behind a feature

**Trigger.** Reviewer objects that shipping a `default = ["emit"]` feature on
runtime makes the "no_std linkable library" claim leaky, or that any user of
`paideia-as-runtime` accidentally trips the std-flavored encoder because they
forgot `default-features = false`.

**Fallback.** Move `emit_instruction` and `EmitError` into a **new**
paper-thin crate `crates/paideia-as-emit/`:

```
paideia-as-emit  →  depends on paideia-as-encoder + paideia-as-runtime
paideia-as-runtime →  no_std, no encoder dep at all
```

Downstream consumers pull `paideia-as-emit` for the runtime API. `emit.rs`
moves verbatim; the tests move verbatim. Runtime keeps its Instruction/IrNodeId
re-exports and stays no_std unconditionally. **LOC impact: +30** (new
Cargo.toml scaffold), same tests. Cleaner but adds a crate boundary that
paideia-os workspace would need to know about. Prefer the feature-gate
approach until reviewer explicitly objects.

## 7.2 The rollback semantics ("buffer unchanged on Err") are deemed too magic

**Trigger.** A reviewer argues that partial emits should be visible to callers
for post-mortem debugging.

**Fallback.** Drop the `buf.bytes.truncate(saved_len)` on the two error paths.
Update the docstring to say "on error the buffer may contain partial bytes;
callers wanting rollback should snapshot `buf.len()` before the call." Change
test #11 to assert `buf.len() >= 2` instead of `== 2`; drop test #12 entirely
and replace with a "reloc detection surfaces before any buffer change" fixture
where we can catch it in pre-flight (fixture #10 already passes pre-flight,
so this remains testable).

**Cost.** −5 LOC in `emit.rs`; strictly weaker safety story for callers. Not
recommended unless review pushback is explicit.

## 7.3 A trait-based `Buf` becomes necessary before r20 closes

**Trigger.** WASM jail (pa-r20-005 example) turns out to want to write directly
into a `&mut [u8]` fixed-size scratch buffer or a raw JIT page, not a
`Vec<u8>`-backed `CodeBuffer`.

**Fallback.** Introduce the trait as a **second overload** in the same
release:

```rust
pub trait EmitBuf {
    fn write_all(&mut self, bytes: &[u8]);
    fn len(&self) -> usize;
    fn truncate(&mut self, len: usize);
}

impl EmitBuf for CodeBuffer { /* … */ }
impl EmitBuf for Vec<u8>    { /* … */ }

pub fn emit_instruction_into<B: EmitBuf>(buf: &mut B, ins: Instruction)
    -> Result<(), EmitError>;
```

`emit_instruction(&mut CodeBuffer, ...)` (concrete) stays as the AC-mandated
signature; `emit_instruction_into<B>` is the escape hatch. **Cost.** +80 LOC,
+3 tests (a `Vec<u8>` fixture and a "custom Buf impl passes" fixture). Push
this to pa-r20-006 or r21 rather than expand r20-002 beyond its 400-LOC
budget — the AC gives no signal that the concrete type is insufficient.

## 7.4 Encoder emits `EncodeOutput` reloc sites for an instruction our pre-flight cleared

**Trigger.** A new encoder-side lowering (e.g., a large-model MOV imm64 that
internally decays into a load-from-literal-pool) starts emitting reloc sites
even from inputs whose operands don't obviously look like relocations. Test
#12's belt-and-braces post-check fires but no pre-flight caught it.

**Fallback.** Neither pre-flight nor post-check is wrong — both are needed.
The one action is to file an encoder issue (pa-r20-NNN) for the specific
mnemonic that surprised us and let the JIT caller either (a) accept the
`UnresolvedRelocation` error and use `resolve_symbols` for that instruction,
or (b) use an alternative operand form that doesn't decay. No design change
here.

## 7.5 `EmitError` is required to be `#[derive(thiserror::Error)]` for downstream tooling

**Trigger.** paideia-os phase-10 code invocation surface uses `thiserror::Error`
in its error chain and wants type-preserving `#[from]` interop.

**Fallback.** Add `thiserror` (workspace pin) to runtime's `[dependencies]`
gated on the `emit` feature. Replace the hand-written `Display` impl with
`#[derive(thiserror::Error)]` attributes. Zero API change, +2 LOC in
Cargo.toml, -20 LOC in emit.rs. Prefer only if paideia-os side actually asks
— hand-written `Display` is one function, and it dodges a proc-macro
dependency in the "linkable library" story.

# 8. Cross-repo (paideia-os) impact
